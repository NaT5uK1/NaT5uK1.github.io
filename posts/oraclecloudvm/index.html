Oracle Cloud VM

创建实例

更改默认镜像为ubuntu20.04，用户名默认为ubuntu

若创建其他镜像，用户名默认为opc

添加SSH密钥对

生成密钥对

  1. 在【实例】-【资源】-【控制台连接】中，【创建本地连接】
  2. 默认选择【为我生成密钥对】
  3. 【保存私有秘钥】【保存公共秘钥】
  4. 将私钥和公钥重命名为oracle oracle.pub（任意合法文件名即可）

管理密钥对

  1. 将oracle oracle.pub复制到~/.ssh中

  2. 为oracle添加权限：

    chmod 400 ~/.ssh/oracle

  3. 由于通常情况下本机已有名为id_rsa id_rsa.pub的密钥对存在，所以需要配置多秘钥管理：

    cd ~/.ssh
vim config

    ~/.ssh/config 配置如下：

    Host host
    HostName host
    IdentityFile ~/.ssh/oracle

Host github.com
    HostName github.com
    User git
    IdentityFile ~/.ssh/id_rsa
    • 上面host需替换为自己的服务器ip或域名

上传公钥

  1. 在【实例】-【资源】-【控制台连接】中，【创建本地连接】
  2. 选择【上载公共秘钥文件(.pub)】
  3. 将oracle.pub拖入秘钥框
  4. 【创建控制台连接】

连接SSH服务

ssh ubuntu@host
  • 提示将host加入known_hosts中按y

更新apt

sudo apt update

安装图形界面xfce4

sudo apt install xfce4 xfce4-goodies

选择gdm3

sudo apt install tightvncserver
  1. 启动vncserver并设置连接密码
vncserver

客户端

  1. 创建一个SSH隧道用于安全地连接到vncserver
ssh -L 59000:localhost:5901 -C -N -l ubuntu host

此时保持客户端shell会话

  1. MacOS下，cmd+space搜索 Screen sharing.app并打开，连接ip地址为localhost:59000，输入vnc连接密码，显示出xfce4图形界面即表示成功，Windows下可以使用PuTTY

将VNC配置为系统服务

服务端

  1. 添加vncserver服务配置文件
sudo vim /etc/systemd/system/vncserver@.service

添加以下配置：

[Unit]
Description=Start TightVNC server at startup
After=syslog.target network.target

[Service]
Type=forking
User=ubuntu
Group=ubuntu
WorkingDirectory=/home/ubuntu

PIDFile=/home/ubuntu/.vnc/%H:%i.pid
ExecStartPre=-/usr/bin/vncserver -kill :%i > /dev/null 2>&1
ExecStart=/usr/bin/vncserver -depth 24 -geometry 1280x800 -localhost :%i
ExecStop=/usr/bin/vncserver -kill :%i

[Install]
WantedBy=multi-user.target
  1. 重新加载单元文件
sudo systemctl daemon-reload
  1. 启动单元文件
sudo systemctl enable vncserver@1.service
  1. 如果vncsever在运行，停止当前服务
vncserver -kill :1
  1. 启动vncserver服务
sudo systemctl start vncserver@1
  1. 验证服务启动状态
sudo systemctl status vncserver@1
